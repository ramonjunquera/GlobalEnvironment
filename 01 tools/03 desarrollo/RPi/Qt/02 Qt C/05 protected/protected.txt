protected

Novedades:
- Uso de etiqueta protected

Descripción:
  En el archivo de cabecera de define una clase: variables y métodos.
  Cada uno de ellos debe estar incluido en una sección.
  
  Acceso          public protected private
  --------------- ------ --------- -------
  Misma clase       SI       SI       SI
  Clase derivada    SI       SI       NO
  Llamada externa   SI       NO       NO

Desarrollo:
  Definimos una clase base con una variable en cada sección.
  El constructor sólo asigna valores a las variables.
  Definimos una clase derivada de la base.
  Definimos un constructor que llama al de la clase base.
  Realmente no es necesario, porque lo hace automáticamente.
  Desde main.cpp se crea una instancia de la clase derivada.
  Se comprueba que se puede acceder directamente a la variable pública de la clase base.
  No se puede acceder de ninguna manera a la variable privada de la clase base.
  Ni desde la clase derivada ni desde una función.
  En la clase derivada hemos creado un método que devuelve el valor de la variable protegida.
  Es la única manera de alcanzarla. No se podria llegar a ella directamente desde la función
  externa.
