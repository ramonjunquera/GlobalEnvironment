Tema: Cómo mapear una unidad de red en Raspbian
Fecha: 20190602

Raspbian incluye por defecto las librerías necesarias para conectar.
Originalmente eran las de Samba (smbclient), pero fueron sustituidas por wheezy.

Lo primero que haremos será crear la carpeta donde montaremos la unidad remota.
En nuestro caso la llamaremos synology y se encontrará en /home/pi/synology
Podríamos utilizar la siguiente secuencia de comandos:
  cd /home/pi
  mkdir synology

A continuación debemos modificar la configuración de las unidades que monta 
cuando arranca. Esto se encuentra en el archivo /etc/fstab
Lo editamos con nano:
  sudo nano /etc/fstab

Añadimos el final del archivo la siguiente línea:
  //192.168.1.197/public /home/pi/synology/ cifs username=usuario,password=contraseña,vers=1.0,rw 0 0
En ella se define:
- La dirección y nombre del recurso compartido
- La carpeta local sobre la que se montará
- El tipo de servidor
- El usuario y contraseña de acceso a la carpeta remota
- Los permisos que se aplicarán por defecto

Guardamos con CTRL+O.
No pregunta si sobreescribimos el archivo. Lo aceptamos.
Salimos con CTRL+X

Para probarlo sólo tenemos que ejecutar:
  sudo mount -a
La carpeta compartida de la unidad de red se ha montado en la carpeta local que
hemos creado.

Si no estamos seguros de la línea escrita en fstab, siempre podemos montar la unidad
manualmente con el siguiente comando:
sudo mount.cifs //192.168.1.197/public /home/pi/synology/ -o username=Ra,password=Amama5,vers=1.0

En el proceso de arranque, si al procesar el archivo fstab no se ha inicializado la red,
las unidades remotas no se montarán.
Para evitar que esto ocurra, debemos obligar al sistema operativo que finalice la
inicialización de la red, antes de procesar fstab.
Esto se hace desde el menú:
  Menú/Preferences/Raspberry Pi Configuration/System/Network at Boot
Y marcar el check de: Wait for network

Es muy conveniente añadir una dirección al apartado Bookmarks para agilizar el salto
a una carpeta concreta.
En las últimas versiones de Raspbian, añadir un bookmark se considera una opción avanzada de
usuario.
Por defecto el sistema viene con el interface simple. Por lo tanto, no aparece la opción.
Para activar las opciones avanzadas de usuario debemos abrir File Manager/Edit/Preferences/Layout
y desmarcar la opción de "Display simplified user interface and menus".
La configuración no tedrá efecto en las sesiones de "File Manager" que estén abiertas.
Sólo en las nuevas.
Ahora al abrir el menú contextual (con el botón derecho) en el panel de carpetas (izquierdo), nos
aparecerá la opción "Add to bookmarks".
El menú principal tendrá la opción "Bookmarks".
Y sobre el panel izquierdo aparecerá el desplegable "Directory Tree" con la posibilidad de seleccionar
"Places" para gestionar los bookmarks.
